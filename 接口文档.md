# 聊天应用后端接口文档

## 🔍 接口

- POST /user/login - 用户登录
- POST /user/register - 用户注册（新增）
- GET /user/user/UserInfo - 获取用户信息（新增）
- POST /user/update - 更新用户信息（新增）
- POST /user/updateAvatar - 更新用户头像（新增）
- POST /character/add - 新增角色
- GET /character/my-list - 获取我的角色列表（更新）
- GET /character/square - 角色广场（新增）
- GET /character/{id} - 获取角色详情
- POST /character/follow - 关注角色（新增）
- POST /user/follow/follow - 关注/取消关注角色（新增）
- GET /user/follow/status - 查看关注状态（新增）
- GET /user/follow/list - 获取关注列表（新增）
- POST /user/common/upload - 通用文件上传
- POST /user/common/uploadCharacterAvatar - 角色头像上传
- POST /user/common/uploadUserAvatar - 用户头像上传（新增）
- GET /ai/chat - AI 聊天

## 项目概述
这是一个基于Spring Boot的聊天应用后端服务，提供用户管理、角色管理、文件上传与 AI 聊天等功能。采用RESTful API设计，使用JWT进行身份认证。

## 技术栈
- 框架: Spring Boot 3.5.5
- 数据库: MySQL 8.0
- 缓存: Redis
- 对象存储: 阿里云OSS
- ORM: MyBatis
- 认证: JWT (JSON Web Token)
- JDK: Java 17

## 基础信息
- 服务端口: 9090
- 基础URL: `http://localhost:9090`
- 字符编码: UTF-8
- 时间格式: yyyy-MM-dd HH:mm:ss
- 跨域支持: 已配置

## 通用响应格式

### 成功响应
```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

### 错误响应
```json
{
  "code": 0,
  "msg": "错误信息",
  "data": null
}
```

### 响应状态码说明
- `code: 1` - 操作成功
- `code: 0` - 操作失败

## 认证机制（更新）
所有需要认证的接口都需要在请求头中添加自定义头：
```
token: <登录返回的 data.token>
```
- 注意：不需要加 `Bearer` 前缀。
- 该头名称来自 application.yml 配置 `user-token-name: token`，拦截器从该头读取令牌。

## 用户管理模块

### 1. 用户登录

- 接口描述: 用户登录认证，成功后返回JWT令牌
- 请求地址: `POST /user/login`
- 请求格式: JSON

请求参数:
| 参数名   | 类型   | 必填 | 说明     |
|----------|--------|------|----------|
| username | string | 是   | 用户名   |
| password | string | 是   | 密码     |

请求示例:
```json
{
  "username": "testuser",
  "password": "123456"
}
```

响应参数:
| 参数名   | 类型   | 说明         |
|----------|--------|--------------|
| id       | number | 用户ID       |
| username | string | 用户名       |
| token    | string | JWT认证令牌  |

响应示例:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "id": 1,
    "username": "testuser",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

## AI 聊天模块（更新）

### 17. 发起聊天（GET）
- 接口描述: 发起一次聊天请求，返回模型纯文本回复；支持传入“角色设定”字段（通过 `Character` 自动绑定）用于注入模板。
- 请求地址: `GET /ai/chat`
- 认证要求: 需要JWT令牌（请求头 `token`）
- 请求参数: Query String

参数说明：

必填参数：
| 参数名      | 类型   | 必填 | 说明                                   |
|-------------|--------|------|----------------------------------------|
| userId      | string | 是   | 用户ID（字符串形式）                   |
| characterId | string | 是   | 角色ID（字符串形式）                   |
| message     | string | 是   | 本次发送的消息内容                     |

可选参数：
| 参数名   | 类型   | 必填 | 说明                                   |
|----------|--------|------|----------------------------------------|
| memoryId | string | 否   | 会话ID；为空时按 `chat_{userId}_{characterId}` 默认会话 |

可选角色设定参数（随 `Character` 绑定，未提供则为空字符串注入）：
| 参数名        | 类型   | 必填 | 说明                                   |
|---------------|--------|------|----------------------------------------|
| name          | string | 否   | 角色名字，注入 {{name}}                |
| appearance    | string | 否   | 外表描述，注入 {{appearance}}          |
| background    | string | 否   | 经历背景，注入 {{background}}          |
| personality   | string | 否   | 性格特征，注入 {{personality}}         |
| classicLines  | string | 否   | 经典台词，注入 {{classic_lines}}       |

- 返回类型: `text/html; charset=UTF-8`（纯文本内容）

请求示例（仅必填参数）：
```bash
curl -G "http://localhost:9090/ai/chat" \
  -H "token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  --data-urlencode "userId=1" \
  --data-urlencode "characterId=assistant" \
  --data-urlencode "message=你好，做个自我介绍"
```

请求示例（带角色设定）：
```bash
curl -G "http://localhost:9090/ai/chat" \
  -H "token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  --data-urlencode "userId=1" \
  --data-urlencode "characterId=assistant" \
  --data-urlencode "message=用你的风格自我介绍一下" \
  --data-urlencode "name=小南" \
  --data-urlencode "appearance=短发，爱穿浅色毛衣，笑容温暖" \
  --data-urlencode "background=曾在公益机构志愿多年，擅长倾听" \
  --data-urlencode "personality=耐心、共情、轻松幽默" \
  --data-urlencode "classicLines=慢慢来，一切都会好起来的。"
```

响应示例（文本）：
```
你好！我是你的 AI 助手，很高兴为你服务~
```

### 18. 获取单个会话历史（GET）
- 接口描述: 获取当前用户与指定角色的默认会话历史（`chat_{userId}_{characterId}`）。
- 请求地址: `GET /ai/chat/history/{userId}/{characterId}`
- 返回类型: `application/json`

路径参数：
| 参数名      | 类型   | 必填 | 说明                 |
|-------------|--------|------|----------------------|
| userId      | string | 是   | 用户ID               |
| characterId | string | 是   | 角色ID               |

响应示例：
```json
[
  { "type": "user", "content": "你好" },
  { "type": "ai",   "content": "你好！我能帮你什么？" }
]
```

说明：服务端会自动调用“会话跟踪”逻辑，若会话首次访问会自动在Redis创建并记录到用户-角色的会话列表中。

## 注意事项

1. 认证令牌管理: Token有效期为2小时，过期后需要重新登录
2. 跨端口访问: 测试页与接口在同端口（9090），fetch 使用相对路径，避免跨域
3. 静态资源放行: 已在 WebMvc 配置中放行静态资源与 /user/login、/error
4. 若图片类型或字体静态资源有新增，请在拦截器放行与资源映射中补充对应后缀

- 新增 AI 会话管理接口：
  - `GET /ai/chat/history/{userId}/{characterId}` 获取默认会话历史
  - `GET /ai/chat/history/session/{sessionId}` 获取指定会话历史
  - `GET /ai/chat/sessions/{userId}/{characterId}` 获取会话列表
  - `POST /ai/chat/new/{characterId}` 新建会话
- 统一并明确测试页的参数与行为说明
